## Governance Purpose

These standards exist to ensure detection content is:
- consistent
- understandable
- maintainable
- reportable
- aligned to repository expectations

## What Governance Covers

Governance in this repository includes:
- naming and file structure
- lifecycle usage
- severity assignment
- tag consistency
- analytic quality expectations
- data source awareness

## How to Use These Standards

When creating or updating detection content:
1. follow the rule template
2. apply the naming standard
3. choose lifecycle intentionally
4. assign severity consistently
5. use tags deliberately
6. ensure the analytic meets quality expectations
