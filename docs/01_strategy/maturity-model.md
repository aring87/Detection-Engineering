# Detection Engineering Maturity Model

## Level 1 — Ad Hoc
- Detections are created reactively
- Little documentation or ownership exists
- No standard lifecycle or validation process
- Coverage is not measurable

## Level 2 — Developing
- Basic standards begin to exist
- Detections are documented in a shared repository
- ATT&CK mapping starts to be applied
- Manual review and tuning occur inconsistently

## Level 3 — Defined
- Standardized schemas, templates, and review processes exist
- Detection lifecycle is documented and followed
- Coverage reporting is maintained
- Triage and tuning processes are established
