# Detection Lifecycle

## Objective

The detection lifecycle defines how analytics move from idea to sustained operational value.

## Lifecycle States

### 1. Intake
A detection opportunity is identified from threat intelligence, incidents, hunts, exercises, coverage gaps, or analyst feedback.

### 2. Design
Define use case, required data sources, ATT&CK mapping, and expected fidelity.

### 3. Build
Author the detection using the repository standard template.

### 4. Validate
Test against benign and malicious examples, historical data, or emulated activity.

### 5. Tune
Refine logic to improve signal quality and reduce operational noise.

### 6. Deploy
Promote through controlled release processes into Sentinel.

### 7. Operate
Monitor alert behavior and capture SOC feedback.

### 8. Measure
Review coverage, quality, and lifecycle metrics regularly.
